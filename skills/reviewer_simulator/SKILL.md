# 审稿专家模拟 Skill

## 概述

本Skill模拟学术审稿专家的视角，对论文进行全面、深入的评审，提供专业的审稿意见和修改建议。

## 适用场景

- 论文投稿前自我评估
- 学位论文送审前模拟评审
- 学术写作训练
- 理解审稿人关注点

## 审稿维度

### 1. 创新性评估 (Novelty)

| 评估等级 | 说明 | 典型意见 |
|---------|------|---------|
| 强创新 | 新方法/新理论/新应用 | Accept |
| 中等创新 | 现有方法的改进或扩展 | Minor Revision |
| 弱创新 | 已有工作的简单应用 | Major Revision |
| 无创新 | 重复已有工作 | Reject |

#### 创新点检查清单

```
□ 与已有方法相比，核心差异是什么？
□ 主要贡献是否清晰明确？
□ 创新是否有理论或实验支撑？
□ 创新是否具有普适性？
□ 创新是否解决了重要问题？
```

### 2. 技术正确性 (Technical Correctness)

| 检查项 | 说明 |
|-------|------|
| 理论推导 | 公式推导是否正确，假设是否合理 |
| 算法描述 | 算法流程是否完整，复杂度分析是否正确 |
| 实验设计 | 对比实验是否公平，指标选择是否恰当 |
| 结果分析 | 结论是否有数据支撑，解释是否合理 |

### 3. 实验充分性 (Experimental Validation)

| 检查项 | 合格标准 |
|-------|---------|
| 对比方法 | 与足够的基线方法对比 |
| 数据集 | 在多个数据集上验证 |
| 消融实验 | 验证各模块的有效性 |
| 统计分析 | 多次实验，报告均值和方差 |
| 失败分析 | 分析方法的局限性和失败案例 |

### 4. 写作质量 (Presentation Quality)

| 检查项 | 说明 |
|-------|------|
| 清晰度 | 论文结构清晰，逻辑连贯 |
| 完整性 | 方法描述完整，可复现 |
| 准确性 | 无拼写错误，术语一致 |
| 简洁性 | 无冗余内容，表达精炼 |

### 5. 文献综述 (Related Work)

| 检查项 | 说明 |
|-------|------|
| 全面性 | 涵盖领域内的重要工作 |
| 时效性 | 包含最新的相关工作 |
| 准确性 | 对他人工作的描述准确 |
| 区分度 | 清晰说明与已有工作的差异 |

## 审稿意见模板

### 总体评价

```markdown
## 总体评价

本文针对[问题描述]，提出了[方法概述]。主要贡献包括：
1. [贡献1]
2. [贡献2]
3. [贡献3]

**推荐意见**: [Accept / Minor Revision / Major Revision / Reject]

**理由**: [简要说明推荐意见的理由]
```

### 详细意见

```markdown
## 详细意见

### 1. 创新性
[评价论文的创新程度]

### 2. 技术正确性
[检查技术细节的正确性]

### 3. 实验设计
[评价实验的充分性和公平性]

### 4. 写作质量
[评价论文的写作水平]

### 5. 文献综述
[评价相关工作的综述质量]
```

### 具体修改建议

```markdown
## 具体修改建议

**必须修改** (Major Issues):
1. [问题描述及修改建议]
2. ...

**建议修改** (Minor Issues):
1. [问题描述及修改建议]
2. ...

**可选修改** (Suggestions):
1. [改进建议]
2. ...
```

## 使用方式

### Python API

```python
from doc_parser import parse_document

# 解析文档
doc = parse_document("manuscript.txt")
content = "\n\n".join([c.content for c in doc.chunks])

# 准备审稿prompt
review_prompt = f"""
请作为审稿专家对以下论文进行评审...

{content}
"""
```

### Kimi Code CLI 中使用

```markdown
请作为一位[机器学习/无人机控制/深度强化学习]领域的审稿专家，
对以下论文进行评审：

论文信息：
- 标题: [论文标题]
- 类型: [学位论文/期刊论文]

[粘贴论文主要内容]

评审维度：
1. 创新性 - 论文的主要贡献和创新点
2. 技术正确性 - 理论推导和算法描述是否正确
3. 实验充分性 - 实验设计是否合理，结果是否有说服力
4. 写作质量 - 论文结构、逻辑、语言表达
5. 文献综述 - 对已有工作的了解程度

请给出：
- 总体评价和推荐意见
- 各维度的详细评价
- 具体的修改建议（必须修改、建议修改、可选修改）
```

## 在 Agent 工作流中使用

```python
# 前置：已完成其他审查Skill

# 收集各维度审查结果
doc = parse_document("manuscript.txt")
content = "\n\n".join([c.content[:3000] for c in doc.chunks[:5]])

# 使用本Skill进行综合审稿评估
review_prompt = f"""
请作为[计算机科学/人工智能]领域的审稿专家，对以下论文进行综合评审：

论文标题：[提取的标题]
摘要：{doc.abstract}

主要内容：
{content}

请从以下维度给出评审意见：
1. 创新性（与现有工作的差异，贡献的明确性）
2. 技术正确性（方法描述的完整性，推导的正确性）
3. 实验充分性（对比方法、消融实验、统计检验）
4. 写作质量（结构清晰度、语言表达）
5. 文献综述（相关工作覆盖度、区分度）

请给出推荐意见（Accept/Minor/Major/Reject）和具体修改建议。
"""

# 输出审稿意见报告
```

## 审稿人角色

### 严格程度

| 级别 | 特点 | 适用场景 |
|-----|------|---------|
| Lenient | 更注重鼓励，对创新容忍度高 | 初稿评估 |
| Normal | 平衡的评价标准 | 投稿前评估 |
| Strict | 注重细节，对缺陷敏感 | 顶会投稿前 |
| Harsh | 非常严格，近乎挑剔 | 高质量要求 |

### 专业领域

- **General**: 通用计算机科学
- **ML/DL**: 机器学习/深度学习
- **RL**: 强化学习
- **CV**: 计算机视觉
- **NLP**: 自然语言处理
- **Robotics**: 机器人学
- **Control**: 控制理论

## 与其他Skill的关系

- **前置Skill**: 所有审查Skill（语法、结构、图表、参考文献、内容）
- **输出**: 综合审稿意见
- **后置Skill**: 写作助手（根据审稿意见修改）
