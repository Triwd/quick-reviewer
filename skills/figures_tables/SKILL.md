# 学术论文图表审查 Skill

## 概述

本Skill专注于学术论文中的图、表规范性检查，包括编号连续性、标题格式、中英文对应、引用准确性等。

## 适用场景

- 学位论文图表审查
- 期刊论文图表规范检查
- 学术报告可视化检查
- 答辩PPT图表质量检查

## 检查维度

### 1. 图表编号检查

| 检查项 | 说明 | 严重程度 |
|-------|------|---------|
| 编号连续性 | 检查是否有跳号（如3.1, 3.2, 3.4缺少3.3） | Critical |
| 编号重复 | 同一编号出现多次 | Critical |
| 章节对应 | 第X章的图编号应为X.Y | Major |
| 顺序一致性 | 文中引用顺序与图表出现顺序一致 | Major |

### 2. 图标题检查

#### 中文图标题

| 检查项 | 规范 | 示例 |
|-------|------|------|
| 格式 | "图X.Y 标题内容" | 图3.1 延迟场景下对局胜率 |
| 简洁性 | 避免过长，一般不超过20字 | - |
| 准确性 | 准确反映图内容 | - |
| 避免口语化 | 不使用"了"、"一下"等 | "嵌入共享Attention" → "基于共享注意力机制" |

#### 英文图标题

| 检查项 | 规范 | 示例 |
|-------|------|------|
| 缩写统一 | 全文统一"Fig."或"Figure" | Fig. 3.1 / Figure 3.1 |
| 大小写 | 推荐Title Case | Fig. 3.1 Win Rate Under Different Delay Conditions |
| 简洁性 | 避免过长 | - |
| 与中文对应 | 语义一致 | - |

#### 常见问题

| 问题类型 | 错误示例 | 正确示例 |
|---------|---------|---------|
| 编号重复 | 图3.7和图3.8英文都为Fig.3.8 | 图3.7→Fig.3.7, 图3.8→Fig.3.8 |
| 中英文不匹配 | 中文"延迟灾难示意" vs 英文"Delay Disaster" | 中文"延迟导致的决策错位示意图" vs 英文"Schematic of Decision Misalignment" |
| 缩写不一致 | 部分用Figure，部分用Fig. | 全文统一为Fig. |
| 大小写混乱 | figure 3.1 或 FIGURE 3.1 | Fig. 3.1 或 Figure 3.1 |
| 缺少内容 | 英文缺少"曲线/Curve" | 补充"Reward Curve" |

### 3. 表标题检查

#### 中文表标题

| 检查项 | 规范 | 示例 |
|-------|------|------|
| 格式 | "表X.Y 标题内容" | 表3.1 交战规则术语定义 |
| 术语准确 | 使用准确的专业术语 | "初始想定参数"而非"超参数" |
| 三线表 | 使用标准三线表格式 | - |

#### 英文表标题

| 检查项 | 规范 | 示例 |
|-------|------|------|
| 格式 | "Table X.Y Title" | Table 3.1 Competition Rules Notation Definitions |
| 大小写 | 推荐Title Case | - |
| 定冠词 | 注意the的使用 | Definition of Core Symbols **in the** Sensor Error Model |

#### 常见问题

| 问题类型 | 错误示例 | 正确示例 |
|---------|---------|---------|
| 编号重复 | 两个表5.1 | 表5.1和表5.2 |
| 术语错误 | "Hyperparameters"（实际为初始状态） | "Parameters in Initialization Scenario" |
| 格式不统一 | Table前空格数不一致 | 统一缩进 |

### 4. 图表引用检查

| 检查项 | 说明 | 严重程度 |
|-------|------|---------|
| 首次引用 | 图表首次出现需有引用 | Major |
| 引用格式 | 统一使用"如图X.Y所示"或"图X.Y" | Minor |
| 引用准确性 | 引用的图表编号正确 | Critical |
| 图文对应 | 正文描述与图表内容一致 | Critical |

### 5. 图表格式检查

#### 图的格式

| 检查项 | 规范 |
|-------|------|
| 清晰度 | 分辨率≥300dpi |
| 字体大小 | 图中文字不小于8pt |
| 颜色使用 | 考虑黑白打印可读性 |
| 坐标轴 | 有标签和单位 |
| 图例 | 清晰可辨，位置合理 |

#### 表的格式

| 检查项 | 规范 |
|-------|------|
| 三线表 | 仅顶线、栏目线、底线 |
| 对齐方式 | 文字左对齐，数字右对齐 |
| 单位标注 | 表头或表注中标注单位 |
| 表注位置 | 表格下方 |

### Kimi Code CLI 中使用

```markdown
请作为学术论文图表审查专家，检查论文中的图、表规范：

[粘贴包含图标题、表标题的内容]

检查项目：
1. **编号连续性**
   - 图/表编号是否连续
   - 有无重复编号
   - 有无跳号

2. **中英文对应**
   - 中英文标题语义是否一致
   - 格式是否统一（Fig. vs Figure）
   - 大小写是否规范

3. **引用准确性**
   - 文中是否正确引用
   - 引用格式是否统一
   - 图文是否对应

4. **格式规范**
   - 图标题格式
   - 表格式（三线表）
   - 中英文间空格

请输出问题清单和修改建议。
```

## 在 Agent 工作流中使用

```python
# 前置：已完成 doc_parser 解析

# 读取包含图表标题的内容
with open("manuscript_content.txt", "r", encoding="utf-8") as f:
    content = f.read()

# 使用正则提取图标题、表标题
import re
figures = re.findall(r'图\s*\d+\.\d+[^\n]*', content)
tables = re.findall(r'表\s*\d+\.\d+[^\n]*', content)

# 使用本Skill进行图表检查

请作为学术论文图表审查专家，检查以下图、表规范：

图标题列表：
{figures}

表标题列表：
{tables}

请检查编号连续性、中英文对应、格式统一性...
"""

# 输出图表审查结果
```

## 预设检查规则

```python
# 图编号正则
FIGURE_PATTERN = r'图\s*(\d+)\.(\d+)'
FIGURE_EN_PATTERN = r'Fig(?:ure)?\.?\s*(\d+)\.(\d+)'

# 表编号正则
TABLE_PATTERN = r'表\s*(\d+)\.(\d+)'
TABLE_EN_PATTERN = r'Table\s+(\d+)\.(\d+)'

# 引用正则
FIGURE_REF_PATTERN = r'(?:如图|图)\s*(\d+)\.(\d+)(?:\s*所示)?'
TABLE_REF_PATTERN = r'(?:如表|表)\s*(\d+)\.(\d+)(?:\s*所示)?'
```
