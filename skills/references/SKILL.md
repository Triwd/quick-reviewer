# 学术论文参考文献审查 Skill

## 概述

本Skill专注于学术论文参考文献的全面审查，包括重复检测、引用匹配、格式规范、引用合理性等。

## 适用场景

- 学位论文参考文献检查
- 期刊论文参考文献审查
- 项目申报书文献核查
- 综述论文文献全面检查

## 检查维度

### 1. 重复文献检测

| 检测类型 | 说明 | 示例 |
|---------|------|------|
| 完全重复 | 同一文献多次出现 | [13]和[30]同为arXiv:1707.06347 |
| 版本重复 | 预印本+正式发表 | [11]Nature版和[22]arXiv版 |
| 作者重复 | 同一作者相似主题多篇 | 需人工判断是否必要 |

### 2. 引用-文献匹配检查

| 检查项 | 说明 | 严重程度 |
|-------|------|---------|
| 引用存在性 | 文中引用的编号在参考文献列表中存在 | Critical |
| 作者匹配 | 引用提及的作者与文献作者一致 | Critical |
| 内容匹配 | 引用内容与文献核心内容一致 | Major |
| 引用完整性 | 提到的研究都有引用标记 | Major |

#### 常见问题

| 问题类型 | 错误示例 | 正确做法 |
|---------|---------|---------|
| 引用错位 | "Mnih[21]等人提出DQN"但[21]是其他文献 | 改为"Mnih[11]" |
| 引用缺失 | "Xia等人利用..."无引用标记 | 添加[57] |
| 引用超出范围 | 引用[100]但只有80条文献 | 修正编号 |

### 3. 引用合理性检查

#### 评估维度

| 维度 | 评估标准 | 低关联度示例 |
|-----|---------|-------------|
| 主题相关性 | 被引文献核心主题与引用内容一致 | 光通信文献→传感器误差 |
| 数据支持 | 具体数据/结论是否来自被引文献 | 气象雷达→雷达测距误差 |
| 方法匹配 | 引用方法是否来自被引文献 | 故障恢复→误差分布模型 |
| 时效性 | 是否使用最新权威文献 | 20年前文献描述现代技术 |

#### 高频引用核查

对于引用频次≥3的文献，需特别核查：

```
检查项：
□ 该文献是否为核心方法/数据来源？
□ 引用位置是否准确反映了文献内容？
□ 是否存在更权威的替代文献？
□ 是否为自引？（如适用）
□ 引用该文献的论述是否准确？
```

### 4. 格式规范检查

#### 常见格式问题

| 问题 | 示例 | 建议 |
|-----|------|------|
| 期刊名大小写 | [11] `nature` vs [43] `Nature` | 统一为首字母大写 |
| 会议名缩写 | [4] 全称 vs [27] 缩写 | 统一使用标准缩写 |
| 日期格式 | [12] `2020-9-15` vs [39] `2024-04-16` | 统一为 `YYYY-MM-DD` |
| 页码格式 | [4]无页码 vs [5] `179-211` | 补充完整页码 |
| 作者格式 | 有的全名，有的缩写 | 统一格式 |

#### GB/T 7714 格式要点

```
期刊：[序号] 作者. 题名[J]. 刊名, 年, 卷(期): 起止页码.
会议：[序号] 作者. 题名[C]//会议名. 出版地: 出版者, 年: 起止页码.
专著：[序号] 作者. 书名[M]. 出版地: 出版者, 年.
学位：[序号] 作者. 题名[D]. 保存地: 保存单位, 年.
专利：[序号] 专利所有者. 专利名称: 专利号[P]. 年.
电子：[序号] 作者. 题名[EB/OL]. (发布日期)[引用日期]. 网址.
```

### 5. 文献类型分布检查

| 类型 | 建议比例 | 说明 |
|-----|---------|------|
| 期刊论文 | 60-70% | 核心期刊为主 |
| 会议论文 | 20-30% | 顶级会议优先 |
| 专著 | 5-10% | 经典教材 |
| 学位论文 | <5% | 必要时引用 |
| 预印本 | <10% | 已发表优先 |

### 6. 时效性检查

| 检查项 | 建议标准 |
|-------|---------|
| 近5年文献 | ≥50% |
| 近3年文献 | ≥30% |
| 经典文献 | 保留关键经典 |
| 过早文献 | 超过20年的需说明必要性 |

## 使用方式

### Python API

```python
from doc_parser import parse_document
import re

# 解析文档
doc = parse_document("manuscript.txt")
content = "\n\n".join([c.content for c in doc.chunks])

# 提取参考文献列表和引用
refs_list = re.findall(r'\[\d+\][^\n]+', content)
citations = re.findall(r'\[(\d+)\]', content)
```

### Kimi Code CLI 中使用

```markdown
请作为学术论文参考文献审查专家，检查参考文献的规范性：

参考文献列表：
[粘贴参考文献列表]

正文引用示例：
[粘贴部分正文引用]

检查项目：
1. **重复文献**
   - 完全重复（相同文献多次出现）
   - 版本重复（预印本+正式发表）

2. **引用-文献匹配**
   - 引用编号与文献是否对应
   - 作者名是否匹配
   - 引用内容与被引文献是否一致

3. **引用合理性**
   - 主题相关性
   - 数据支持
   - 时效性

4. **格式规范**
   - 是否符合GB/T 7714
   - 期刊名大小写
   - 日期格式
   - 页码完整性

5. **文献分布**
   - 近5年文献比例
   - 文献类型分布

请输出详细的问题清单和修改建议。
```

with open("manuscript_content.txt", "r", encoding="utf-8") as f:
    content = f.read()

# 使用正则提取参考文献列表
import re
ref_section = re.search(r'参考文献[\s\S]*?(?=致谢|附录|$)', content)
