# 学术论文结构审查 Skill

## 概述

本Skill专注于学术论文的结构合规性检查，包括章节完整性、标题层级规范、逻辑连贯性以及必要组成部分检查。

## 适用场景

- 学位论文结构检查
- 期刊论文格式审查
- 开题报告/答辩PPT结构检查
- 项目申报书结构审查

## 检查维度

### 1. 必备组成部分检查

#### 学位论文必备部分

| 部分 | 检查内容 | 常见问题 |
|-----|---------|---------|
| 封面 | 题目、作者、导师、单位、日期 | 格式不统一、信息缺失 |
| 中英文摘要 | 独立成页、字数合规 | 中英文不一致、超字数 |
| 目录 | 页码正确、层级清晰 | 页码错误、标题缺失 |
| 图表清单 | 单独成页、编号连续 | 未单独列出、编号跳号 |
| 符号说明 | 公式符号、缩写词 | 符号未说明、前后矛盾 |
| 正文 | 章节完整、逻辑连贯 | 缺少小结、章节跳跃 |
| 参考文献 | 数量达标、格式统一 | 数量不足、格式混乱 |
| 致谢 | 独立成页、内容得体 | 与摘要雷同 |

#### 期刊论文必备部分

| 部分 | 检查内容 |
|-----|---------|
| 标题 | 简洁明了，突出创新点 |
| 作者信息 | 单位、邮箱、通讯作者 |
| 摘要 | 结构完整（背景-方法-结果-结论）|
| 关键词 | 3-5个，涵盖核心概念 |
| 引言 | 研究背景、现状、问题、目标 |
| 方法 | 可复现的技术细节 |
| 结果 | 数据支撑，图表清晰 |
| 讨论 | 结果解释，与已有工作对比 |
| 结论 | 主要发现，局限与展望 |
| 参考文献 | 格式符合期刊要求 |

### 2. 章节结构检查

#### 标题层级规范性

```
标题结构检查流程
├── 1级（章标题）检查
│   ├── 概括性：能否概括全章核心内容
│   ├── 简洁性：字数控制在15字以内
│   └── 一致性：术语与全文保持统一
├── 2级（节标题）检查
│   ├── 内容对应：与实际内容匹配
│   ├── 逻辑顺序：符合认知/研究逻辑
│   └── 格式规范：无重复文字、无语病
├── 3级（小节标题）检查
│   ├── 层次分明：与上下级关系清晰
│   ├── 编号正确：层级编号无错位
│   └── 避免过深：一般不建议超过3级
└── 4级+ 检查
    ├── 必要性：是否必须细分
    └── 内容充实：有足够内容支撑
```

#### 常见问题

| 问题类型 | 错误示例 | 正确示例 |
|---------|---------|---------|
| 文字重复 | 课程学习**理论理论**与算法 | 课程学习理论与算法 |
| 术语不一致 | 第4章用"飞行器"，其他用"无人机" | 全文统一用"无人机" |
| 标题过长 | 面向非完备信息战场态势的空战机动决策算法（19字） | 面向复合非完备环境的无人机机动决策算法（16字） |
| 内容概括不准确 | 面向时间延迟环境的机动决策算法 | 面向时间延迟**非完备**环境的机动决策算法 |
| 编号层级混乱 | 标题中包含"与4.5.1.2节"字样 | 清理标题内容 |

### 3. 逻辑连贯性检查

#### 章节间逻辑

| 检查点 | 说明 | 检查方法 |
|-------|------|---------|
| 引言-结论呼应 | 研究目标与结论是否对应 | 对比阅读 |
| 方法-结果对应 | 方法描述与实验结果匹配 | 交叉核对 |
| 问题-方案对应 | 提出的问题是否有解决方案 | 逐条检查 |
| 承上启下 | 章节过渡是否自然 | 阅读流畅度 |

#### 段落内逻辑

| 检查点 | 说明 |
|-------|------|
| 主题句明确 | 每段首句概括段落主旨 |
| 论证充分 | 论点有数据/文献支撑 |
| 结论清晰 | 段落结尾有总结或过渡 |

# 解析文档
doc = parse_document("manuscript.txt")

# 提取章节结构
chapters = [c['title'] for c in doc.chapters]

### Kimi Code CLI 中使用

```markdown
请作为学术论文结构审查专家，检查以下论文的结构规范性：

论文信息：
- 类型: [学位论文/期刊论文]
- 章节大纲:
[粘贴章节结构]

请检查：
1. **必备组成部分**
   - 是否包含所有必要章节
   - 章节顺序是否正确

2. **标题层级规范性**
   - 标题是否超过4级
   - 是否存在文字重复
   - 标题长度是否适中
   - 术语是否一致

3. **逻辑连贯性**
   - 引言与结论是否呼应
   - 章节间过渡是否自然
   - 问题-方案是否对应

请输出：
- 严重问题列表（必须修改）
- 中等问题列表（建议修改）
- 修改建议对照表
```

## 论文类型模板

### 学位论文模板 (thesis)

```python
THESIS_TEMPLATE = {
    "required_sections": [
        "封面", "声明", "摘要", "Abstract", 
        "目录", "第1章 引言", "结论与展望",
        "参考文献", "致谢"
    ],
    "optional_sections": [
        "图表清单", "符号说明", "攻读学位期间成果"
    ],
    "chapter_pattern": r"第[一二三四五六七八九十\d]+章",
    "max_title_length": {
        1: 20,  # 章标题
        2: 25,  # 节标题
        3: 30   # 小节标题
    }
}
```
