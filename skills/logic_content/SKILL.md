# 学术论文逻辑与内容审查 Skill

## 概述

本Skill专注于学术论文的逻辑严密性、内容完整性、论证充分性以及学术规范性审查。

## 适用场景

- 学位论文内容质量审查
- 期刊论文学术质量评估
- 科研项目结题报告审查
- 学术论文投稿前评估

## 检查维度

### 1. 摘要质量检查

| 检查项 | 说明 | 合格标准 |
|-------|------|---------|
| 结构完整性 | 背景-问题-方法-结果-结论 | 五要素齐全 |
| 信息一致性 | 中英文摘要内容对应 | 无重大差异 |
| 独立性 | 不引用文献、不出现公式 | 独立成篇 |
| 关键词 | 3-5个，涵盖核心概念 | 准确概括主题 |

#### 常见问题

| 问题 | 错误示例 | 正确做法 |
|-----|---------|---------|
| 缺少结果 | 只写方法不写实验结果 | 补充定量结果 |
| 过于笼统 | "取得了良好效果" | 给出具体指标 |
| 中英文不一致 | 中文有3点创新，英文2点 | 逐句对照 |

### 2. 引言逻辑检查

#### 引言应包含的要素

```
引言逻辑链条
├── 研究背景
│   └── 从宏观到微观，引入研究领域
├── 研究现状（文献综述）
│   ├── 述：已有研究做了什么
│   └── 评：已有研究的不足
├── 研究问题
│   └── 明确指出本文要解决什么问题
├── 研究内容
│   └── 概括本文的主要工作
└── 论文结构
    └── 介绍各章节安排
```

#### 检查点

| 检查点 | 说明 | 常见问题 |
|-------|------|---------|
| 文献综述深度 | 对关键文献的分析评价 | 简单罗列，缺乏评述 |
| 问题引出自然 | 从现状不足引出研究问题 | 突兀提出研究问题 |
| 创新点预告 | 引言末尾概括主要贡献 | 创新点不明确 |

### 3. 方法描述检查

#### 方法章节应包含

| 要素 | 说明 | 检查标准 |
|-----|------|---------|
| 问题定义 | 明确数学模型或形式化定义 | 清晰无歧义 |
| 算法流程 | 清晰的步骤描述 | 可复现 |
| 关键推导 | 核心公式的推导过程 | 逻辑严密 |
| 复杂度分析 | 时间/空间复杂度 | 有定量分析 |

#### 常见问题

| 问题 | 说明 | 严重程度 |
|-----|------|---------|
| 符号不统一 | 同一符号不同含义 | Critical |
| 公式错误 | 推导错误或符号错误 | Critical |
| 描述不完整 | 关键步骤缺失 | Major |
| 伪代码缺失 | 算法没有形式化描述 | Major |

### 4. 实验设计检查

#### 实验应包含的要素

| 要素 | 检查内容 |
|-----|---------|
| 环境设置 | 硬件配置、软件版本、数据集 |
| 对比方法 | 基线方法选择依据 |
| 评价指标 | 指标定义、计算公式 |
| 参数设置 | 超参数取值及设置依据 |
| 消融实验 | 各模块贡献分析 |
| 统计显著性 | 多次实验、方差分析 |

#### 常见问题

| 问题 | 说明 | 建议 |
|-----|------|------|
| 对比不公平 | 使用不同训练轮数 | 统一实验条件 |
| 指标不明确 | 自定义指标未定义 | 补充定义和计算方式 |
| 缺少统计检验 | 单次实验结果 | 多次重复取均值方差 |
| 消融不完整 | 缺少关键模块分析 | 补充消融实验 |

### 5. 结果分析检查

| 检查点 | 说明 |
|-------|------|
| 结果解释 | 不仅描述现象，还要解释原因 |
| 对比分析 | 与基线方法的详细对比 |
| 案例展示 | 典型成功案例和失败案例分析 |
| 局限性讨论 | 方法的适用边界 |

### 6. 结论质量检查

| 检查点 | 说明 | 常见问题 |
|-------|------|---------|
| 非简单重复 | 不是摘要或引言的复制 | 避免直接复制粘贴 |
| 创新点总结 | 明确列出主要贡献 | 创新点模糊 |
| 局限性与展望 | 客观分析不足 | 只谈成绩不谈问题 |

### 7. 学术规范性检查

| 检查项 | 说明 |
|-------|------|
| 术语一致性 | 全文术语统一 |
| 缩写规范 | 首次出现全称+缩写 |
| 公式编号 | 连续编号，正确引用 |
| 图表引用 | 文中引用与图表对应 |
| 文献引用 | 引用与内容匹配 |

## 使用方式

### Python API

```python
from doc_parser import parse_document

# 解析文档
doc = parse_document("manuscript.txt")

# 提取各章节内容
abstract = doc.abstract
chapters_content = "\n\n".join([c.content for c in doc.chunks])
```

### Kimi Code CLI 中使用

```markdown
请作为学术内容审查专家，审查论文的学术质量：

[粘贴论文主要内容：摘要、引言、方法、实验、结论]

审查维度：

1. **摘要质量** (20分)
   - 五要素是否齐全：背景、问题、方法、结果、结论
   - 中英文是否一致
   - 是否包含具体定量结果

2. **引言逻辑** (20分)
   - 背景→现状→问题→内容→结构
   - 文献综述是否有评述
   - 问题引出是否自然

3. **方法描述** (20分)
   - 问题定义是否清晰
   - 算法流程是否完整
   - 关键推导是否正确
   - 复杂度分析

4. **实验设计** (20分)
   - 对比方法是否充分
   - 评价指标是否明确
   - 消融实验是否完整
   - 统计显著性

5. **结论质量** (20分)
   - 是否准确概括创新点
   - 是否讨论局限性

请给出各维度得分、主要问题和改进建议。
```

## 学术质量评估标准

| 维度 | 优秀(90-100) | 良好(80-89) | 中等(70-79) | 待改进(<70) |
|-----|-------------|------------|-----------|-----------|
| 摘要 | 五要素齐全，定量具体 | 要素齐全，略欠具体 | 缺少1个要素 | 缺少多个要素 |
| 引言 | 逻辑严密，评述深入 | 逻辑清晰，评述充分 | 有逻辑，评述一般 | 逻辑混乱 |
| 方法 | 完整可复现 | 基本完整 | 有缺失 | 描述不清 |
| 实验 | 设计科学，分析深入 | 设计合理，分析充分 | 基本合理 | 有明显缺陷 |
| 结论 | 创新点明确 | 概括较好 | 与摘要重复 | 内容空洞 |
