# 文档解析 Skill

## 概述

本Skill指导大模型如何对学术论文文档进行解析和分片处理，以便进行后续的审查工作。

## 支持格式

| 格式 | 扩展名 | 处理方式 |
|------|--------|----------|
| 纯文本 | `.txt` | 直接读取 |
| Word文档 | `.doc`, `.docx` | 使用可用工具提取文本 |
| PDF文档 | `.pdf` | 使用可用工具提取文本 |
| Markdown | `.md` | 直接读取 |

## 解析原则

### 1. 大文件分片解析

> ⚠️ **重要提示**：当论文字数超过模型上下文限制时，必须采用分片解析策略。

**分片策略：**

```
单篇论文字符数 > 20,000 字  →  必须分片
单篇论文字符数 > 50,000 字  →  建议分多轮解析
```

**分片方法：**

1. **按章节分片**（推荐）
   - 将论文按自然章节拆分（引言、方法、实验、结论等）
   - 每个章节独立进行审查
   - 保持章节内的逻辑完整性

2. **按段落分片**
   - 当单章节过长时，按段落进一步拆分
   - 确保分片边界在段落结束处，避免断句

3. **重叠区域**
   - 相邻分片之间保留约500字符的重叠区域
   - 确保跨分片的上下文不被割裂

### 2. 元数据提取

解析时应优先提取以下元数据：

| 元数据 | 说明 | 提取位置 |
|--------|------|----------|
| 标题 | 中英文标题 | 文档开头 |
| 摘要 | 中英文摘要 | 标题之后 |
| 关键词 | 中英文关键词 | 摘要之后 |
| 章节结构 | 各级标题列表 | 目录或正文 |
| 图表数量 | 图/表总数统计 | 全文扫描 |
| 参考文献数 | 文献条目数 | 参考文献章节 |

### 3. 内容保留原则

解析时应保留以下内容：

- ✅ 完整的章节文本
- ✅ 图表标题（图X.Y / 表X.Y）
- ✅ 公式标记（公式(X)）
- ✅ 参考文献引用标记（[X]）
- ⚠️ 公式内容（保留LaTeX或文本形式）

## 使用方式

### 单文件解析（文件较小时）

当论文总字数 < 20,000 字符时：

1. 一次性读取完整文档
2. 提取元数据（标题、摘要、关键词）
3. 直接进行后续审查

### 分片解析（文件较大时）

当论文总字数 >= 20,000 字符时：

```
第1轮：提取元数据
        ├── 标题
        ├── 摘要
        ├── 关键词
        └── 章节大纲

第2-N轮：逐章审查
        ├── 第1章：引言 + 相关工作
        ├── 第2章：方法描述
        ├── 第3章：实验设计
        ├── 第4章：结果分析
        └── 第5章：结论 + 参考文献
```

**分片审查的整合：**

每轮分片审查后，应：
1. 记录该章节的审查结果
2. 更新全局问题清单
3. 注意跨章节的关联问题

## 审查工作流示例

### 完整解析流程

```
用户输入：审查论文 manuscript.doc

步骤1：评估文件大小
    ├── 小文件（<20K字）→ 完整解析
    └── 大文件（>=20K字）→ 分片解析

步骤2：提取元数据（所有文件）
    ├── 标题
    ├── 摘要
    ├── 关键词
    └── 章节结构

步骤3：分片内容审查
    ├── 分片1：引言 + 相关工作
    ├── 分片2：方法描述
    ├── 分片3：实验设计
    └── ...

步骤4：整合审查结果
    ├── 汇总所有分片发现的问题
    ├── 识别跨章节的关联问题
    └── 生成最终审查报告
```

## 分片解析的Prompt模板

### 元数据提取Prompt

```markdown
请提取以下论文的元数据：

[论文内容或论文路径]

请提取并返回：
1. 中文标题
2. 英文标题
3. 中文摘要
4. 英文摘要
5. 关键词
6. 章节大纲（列出所有章节标题）
7. 总字数估算

返回格式：JSON
```

### 分片审查Prompt

```markdown
请审查论文的以下章节：

章节：第X章 XXXXX
内容：
[章节内容]

上下文信息：
- 论文标题：[标题]
- 本章节在论文中的位置：[位置]
- 前一章节主题：[主题]
- 后一章节主题：[主题]

请检查：
1. 语法和拼写错误
2. 术语一致性
3. 图表引用
4. 逻辑连贯性
5. 与前后章节的衔接

返回格式：结构化问题列表
```

## 注意事项

### 分片边界处理

- **避免在句子中间截断**：分片边界应在段落结束处
- **保持引用完整性**：确保引用标记[1]不被截断
- **保留章节标题**：每个分片应包含其章节标题

### 跨分片问题识别

以下问题可能需要跨分片检查：

| 问题类型 | 说明 | 处理方式 |
|---------|------|----------|
| 术语一致性 | 不同章节术语是否统一 | 维护全局术语表 |
| 引用匹配 | 引用编号与参考文献对应 | 全局引用检查 |
| 图表编号 | 编号是否连续 | 全局图表清单 |
| 公式编号 | 公式引用是否正确 | 全局公式清单 |

### 文件编码处理

- 优先尝试 UTF-8 编码
- 中文文档可能使用 GBK/GB2312 编码
- 保留原始文档的换行和段落结构
