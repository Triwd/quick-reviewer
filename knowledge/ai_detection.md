# 学术论文AI痕迹检测 Skill

## 概述

本Skill专注于检测学术论文中可能存在的AI生成痕迹，包括语言模式分析、风格一致性检查以及典型AI生成特征识别。

## 适用场景

- 学位论文AI痕迹自查
- 期刊论文投稿前检查
- 学术不端预防
- 写作风格优化

## 检测维度

### 1. 语言模式分析

#### 典型AI生成特征

| 特征 | 说明 | 示例 |
|-----|------|------|
| 过度使用过渡词 | 频繁使用"此外"、"然而"、"因此" | 每段开头都是"此外" |
| 句式单一 | 句子长度和结构过于一致 | 多为"主谓宾"简单句 |
| 缺乏具体细节 | 描述过于笼统抽象 | "取得了良好效果" |
| 模板化表达 | 使用常见AI模板 | "随着XX的发展..." |
| 语义重复 | 同一意思换说法重复 | 不同段落表达相似 |

#### AI常用过渡词

```
高频AI过渡词：
- 首先/其次/最后
- 此外/另外/除此之外
- 然而/但是/尽管如此
- 因此/所以/综上所述
- 值得注意的是/需要指出的是
- 从某种意义上说/总体而言
```

### 2. 风格一致性检查

| 检查项 | 说明 |
|-------|------|
| 人称一致性 | 全文人称统一（本文/我们/笔者） |
| 语体一致性 | 学术语体与口语区分 |
| 段落长度分布 | 段落长度是否过于均匀 |

### 3. 文本统计分析

| 指标 | 正常范围 | AI生成倾向 |
|-----|---------|-----------|
| 平均句长 | 15-25字 | 过于一致或过长 |
| 词汇多样性 | 0.6-0.8 | 过高（词汇堆砌） |
| 重复率 | <10% | 段落间重复高 |
| 标点分布 | 自然分布 | 过于规律 |

## 使用方式

### Python API

```python
from doc_parser import parse_document

# 解析文档
doc = parse_document("manuscript.txt")
content = "\n\n".join([c.content for c in doc.chunks])

# 进行文本统计分析
import re
sentences = re.split(r'[。！？.!?]', content)
avg_sentence_length = sum(len(s.strip()) for s in sentences) / len(sentences)
```

## 风险等级定义

| 得分 | 风险等级 | 说明 | 建议 |
|-----|---------|------|------|
| 0-20 | Low | AI痕迹不明显 | 正常提交 |
| 21-40 | Medium | 存在部分AI特征 | 建议人工润色 |
| 41-60 | High | AI特征较明显 | 需要大幅修改 |
| 61-100 | Very High | 高度疑似AI生成 | 必须重写 |

## 改进建议

### 降低AI痕迹的方法

1. **增加个人风格**
   - 使用自己习惯的表达方式
   - 加入个人对问题的理解和思考

2. **具体化描述**
   - 用具体数据替代笼统表述
   - 增加案例和细节

3. **多样化句式**
   - 长短句结合
   - 适当使用复杂句式

4. **自然过渡**
   - 减少机械化的过渡词
   - 增加段落间的逻辑关联

5. **深入分析**
   - 不仅描述现象，还要解释原因
   - 加入批判性思考
