# 学术写作助手 Skill

## 概述

本Skill作为学术写作助手，根据审稿意见提供修改建议，协助作者改进论文质量，包括润色、重构、补充等全方位写作支持。

## 适用场景

- 根据审稿意见修改论文
- 提升论文语言表达质量
- 优化论文结构和逻辑
- 补充和完善论文内容

## 功能模块

### 1. 语言润色

#### 润色维度

| 维度 | 说明 | 示例 |
|-----|------|------|
| 语法修正 | 纠正语法错误 | "研究围绕" → "本研究围绕" |
| 用词优化 | 更准确的学术表达 | "好的" → "有效的" |
| 句式改进 | 更流畅的句子结构 | 长句拆分、短句合并 |
| 风格统一 | 保持全文风格一致 | 人称、时态统一 |

#### 润色指令

```markdown
请润色以下段落，要求：
1. 修正语法错误
2. 使用更准确的学术术语
3. 保持句子流畅
4. 保持原意不变

原文：[待润色文本]
```

### 2. 结构优化

#### 结构改进

| 问题类型 | 解决方案 |
|---------|---------|
| 章节逻辑不清 | 重新组织章节顺序，增加过渡段落 |
| 内容分布不均 | 调整各章节篇幅，突出重点 |
| 缺少必要内容 | 补充引言、相关工作、结论等 |
| 重复冗余 | 删除重复内容，精简表达 |

#### 重构指令

```markdown
请重构以下章节，要求：
1. 明确段落主旨
2. 增加段落间的逻辑连接
3. 突出核心贡献
4. 删除冗余内容

原文：[待重构文本]
```

### 3. 内容补充

#### 补充类型

| 类型 | 说明 |
|-----|------|
| 相关工作 | 补充最新的相关研究 |
| 实验分析 | 补充消融实验、对比实验 |
| 理论推导 | 补充详细的公式推导 |
| 案例分析 | 补充具体的应用案例 |

#### 补充指令

```markdown
请补充以下内容：
类型：[相关工作/实验/理论/案例]
要求：
1. [具体要求1]
2. [具体要求2]

上下文：[相关上下文]
```

### 4. 审稿回复辅助

#### 回复模板

```markdown
## 对审稿意见的回复

### 意见1: [审稿意见摘要]

**审稿人原文**:
> [审稿意见原文]

**回复**:
[对意见的回应说明]

**修改**:
- 位置: [章节/段落]
- 原文: [原文内容]
- 修改后: [修改后内容]
```

#### 常见回复策略

| 意见类型 | 回复策略 |
|---------|---------|
| 误解 | 礼貌澄清，引用原文说明 |
| 正确 | 接受并详细说明修改 |
| 部分正确 | 接受合理部分，解释不同意的部分 |
| 无法修改 | 解释原因，提供替代方案 |

## 使用方式

### Python API

```python
from doc_parser import parse_document

# 解析文档
doc = parse_document("manuscript.txt")

# 读取需要修改的章节
chunk = doc.chunks[0]
original_text = chunk.content

# 准备润色/修改prompt
polish_prompt = f"""
请润色以下段落...

原文：{original_text}
"""
```

## 在 Agent 工作流中使用

```python
# 前置：已完成审稿模拟，获得审稿意见

# 场景1: 语言润色
doc = parse_document("manuscript.txt")
content = doc.chunks[0].content

polish_prompt = f"""
请作为学术写作助手，润色以下段落：

原文：
{content}

要求：
1. 修正语法错误
2. 使用更准确的学术术语
3. 保持句子流畅
4. 保持原意不变

请输出：
- 修改后的文本
- 修改说明（列出主要修改点）
"""

# 场景2: 根据审稿意见修改
review_comment = "引言部分对研究问题的描述不够清晰"
original_text = "[引言段落]"

revise_prompt = f"""
请根据以下审稿意见修改论文：

**审稿意见**：{review_comment}

**原文**：
{original_text}

请给出：
1. 修改后的文本
2. 修改说明（如何回应该审稿意见）
"""

# 场景3: 生成审稿回复
review_comment = "缺少与SOTA方法的对比"
our_response = "我们已在第4章补充了与XX方法的对比实验"

reply_prompt = f"""
请帮我撰写对以下审稿意见的回复：

**审稿意见**：{review_comment}

**我们的回应**：{our_response}

请生成正式的回复信段落，包括：
1. 对审稿人意见的感谢
2. 我们的回应说明
3. 具体的修改内容
"""
```

## 修改建议格式

### 语言润色输出

```json
{
  "original": "研究围绕非完备信息环境下的无人机机动决策问题...",
  "polished": "本研究围绕非完备信息环境下的无人机机动决策问题...",
  "changes": [
    {
      "type": "grammar",
      "original": "研究围绕",
      "change": "本研究围绕",
      "reason": "缺少主语"
    }
  ],
  "explanation": "补充主语使句子完整，符合学术写作规范"
}
```

### 结构优化输出

```json
{
  "original_structure": ["引言", "方法", "实验", "结论"],
  "suggested_structure": ["引言", "相关工作", "方法", "实验", "结论"],
  "changes": [
    {
      "type": "add",
      "content": "相关工作",
      "position": "引言之后",
      "reason": "缺少文献综述，无法体现与已有工作的区别"
    }
  ],
  "restructured_text": "[重构后的文本]"
}
```

### 审稿回复输出

```json
{
  "replies": [
    {
      "comment_id": 1,
      "comment_summary": "缺少与SOTA的对比",
      "reply_text": "感谢审稿人的宝贵意见。我们已在第4章补充了与XX方法的对比实验...",
      "modifications": [
        {
          "location": "第4.2节",
          "original": "...",
          "revised": "..."
        }
      ]
    }
  ],
  "cover_letter": "[完整的回复信]"
}
```

## 润色风格

| 风格 | 说明 | 适用场景 |
|-----|------|---------|
| Academic | 正式学术风格 | 学位论文、期刊论文 |
| Concise | 简洁风格 | 会议论文、短篇 |
| Detailed | 详细风格 | 技术报告、综述 |
| Native | 母语化表达 | 英文论文润色 |
